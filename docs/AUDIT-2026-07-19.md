## Audit round — 2026-07-23 (v0.1.9.2: rate-limit playback precedence, version, category rename, nav scroll)

R1 (adversarial): 0 Critical (no auth/limit bypass), 2 Warnings. The headline
fix: the global rate limiter (was `max:120/min` on EVERY route) 429'd playback
because HLS issues one request per segment; now the media data plane is exempt
via `allowList`/`isDataPlanePath` and the control-plane ceiling is raised +
configurable (`VLMP_RATE_LIMIT_MAX`, default 600, floored at 60). R1 caught that
the first exemption `/^\/stream\//` was TOO broad — it un-limited the session-
lifecycle POSTs `/stream/:id/start` + `/stream/guest/:code/start`, and since a
DIRECT-PLAY session has no cap (only transcodes hit maxTranscodeSessions), that
was an unbounded session-Map-growth DoS. Fixed: exemption narrowed to delivery
only — `/stream/**.(m3u8|ts)`, `/stream/:id/direct`, thumbnails, subtitle files,
federated HLS — keeping start/keepalive/stop/sessions limited. Also verified:
env parse can't disable/under-set the limiter (floor 60); readAppVersion is
path-traversal-safe and normalizes `0.1.9-2`→`0.1.9.2`; PATCH category rename is
slug-immutable, parameterized, adminOnly, and can't orphan folders (they key on
slug).

R2 (fix-the-fix): FAIL→fixed. The narrowed federation pattern covered federated
HLS (`.m3u8`/`.ts`) but NOT federated DIRECT play (`/federation/servers/:id/
stream/:sid/direct`), asymmetric with local direct — it would have been re-
throttled. Added the federation-direct pattern + test. Re-verdict: PASS.

Verified live on the compiled build (flood test, `VLMP_RATE_LIMIT_MAX=60`, 70
reqs each): control-plane `/library/browse` 429s past 60; segment `.ts` and
`master.m3u8` never 429 (exempt); `/stream/:id/start` stays limited; `/version`
returns `0.1.9.2`. Tests 361→366.
